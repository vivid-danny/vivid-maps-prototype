# Map Data Pipeline

This document captures the state of the map data quality work — where the pipeline stands, what the correction workflow looks like, and what's next.

---

## Overall Goal

The GeoJSON generated by the current pipeline is geometrically sloppy — section polygons are approximate rectangles that don't conform to the actual curved venue shape, and zone/seat placement has inaccuracies. The goal is to establish a **Figma-based correction loop** that lets us manually clean up the geometry, then feed corrected data back into the prototype.

Long-term this should inform what the pipeline needs to produce: accurate, clean GeoJSON that MapLibre can render faithfully without manual correction.

---

## Pipeline Assets (per venue)

| File | What it is |
|------|-----------|
| `public/sections.geojson` | Section polygons (97 features, IDs 101–334 + misc) |
| `public/rows.geojson` | Row polygons nested inside sections |
| `public/seats.geojson` | Individual seat points |
| `public/venue-chrome.geojson` | Arena boundary + court + bench (vector) |
| `public/background.png` | Raster stadium illustration (used as MapLibre image source) |
| `public/manifest.json` | Bounds, center, section centers, row centers |

The coordinate system is synthetic lat/lng near origin — all values are small decimals (lng ≈ 0.0003–0.0986, lat ≈ −0.0726 to −0.0010).

## Correction Workflow

### 1. Edit in Figma
- Import `public/venue-map.svg` into Figma
- Import `public/background.png` as a reference layer behind the SVG at reduced opacity
- Correct section shapes to fit the actual stadium geometry
- Fix zone color assignments if needed
- Delete or correct the venue chrome vectors

### 2. Export back as SVG
- Export the corrected file as SVG from Figma
- Section IDs are preserved as path IDs (`id="section-101"`) — the mapping back to GeoJSON is unambiguous

### 3. Convert back to GeoJSON
- Parse the exported SVG paths
- Reverse the coordinate projection (SVG px → synthetic lat/lng)
- Update `sections.geojson` with corrected coordinates
- Recompute `manifest.json` section centers from the new polygons

> **Note:** The projection formula used in `public/venue-map.svg`:
> ```
> svgX = offsetX + (lng - minX) * scale
> svgY = offsetY + (maxY - lat) * scale   ← Y is flipped
> ```
> Where `scale = min(scaleX, scaleY)`, canvas is 1200×900, padding is 40px each side.

### Regenerating the SVG
```bash
node -e "$(cat scripts/generate-venue-svg.js)"
```

## Known Issues in Current GeoJSON

- `BANNER LOUNGE Compound` and `VISITORS Compound` sections have invalid/empty geometry (NaN centroids) — skip or handle separately
- Section polygons are rectangular approximations — don't follow the curved tiers of the stadium
- Some sections may be in the wrong radial position relative to the actual stadium layout
